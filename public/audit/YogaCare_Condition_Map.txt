============================
EXERCISE TAGGING LOGIC
============================

Each MasterExercise has a `tags` object with boolean safety flags:
  - universalSafe: boolean     — safe for all general conditions
  - pregnancySafe: boolean     — safe during pregnancy
  - discSafe: boolean          — safe for disc herniation / sciatica
  - kneeSafe: boolean          — safe for knee / hip conditions
  - oaSafe: boolean            — safe for osteoarthritis
  - shoulderSafe: boolean      — safe for neck / shoulder conditions
  - flareSafe: boolean         — safe during flare-ups

CONDITION → SAFETY TAG MAPPING:
  fibromyalgia       → universalSafe
  back_pain          → universalSafe
  neck_pain          → shoulderSafe
  shoulder_pain      → shoulderSafe
  knee_pain          → kneeSafe
  hip_pain           → kneeSafe
  pregnancy          → pregnancySafe
  postpartum         → pregnancySafe
  disc_herniation    → discSafe
  sciatica           → discSafe
  osteoarthritis     → oaSafe
  post_injury_rehab  → universalSafe
  stress_anxiety     → universalSafe
  sleep_issues       → universalSafe
  menopause          → universalSafe
  older_adult        → universalSafe
  cross_training     → universalSafe
  general_yoga       → universalSafe

An exercise is included for a condition ONLY if its corresponding safety tag is true.
For flare mode, flareSafe must also be true.

============================
ADDITIONAL SAFETY FILTERING
============================

1. Disc/Sciatica Contraindication Filter:
   If user has disc_herniation OR sciatica:
   - If exercise has contraindicationFlags.avoidFlexion = true → EXCLUDED
   - If exercise has contraindicationFlags.avoidRotation = true → EXCLUDED
   - Fallback: if no contraindicationFlags, check contraindications[] array
     for strings containing "disc", "sciatica", "flexion", or "rotation"

2. Pregnancy Override:
   If user has pregnancy condition:
   - Exercise must have pregnancySafe = true (checked AFTER general safety tag)

============================
RECOMMENDATION ENGINE SUMMARY
============================

1. EXERCISE SELECTION PIPELINE:
   a. Filter: All master exercises are filtered by safety tags for the user's conditions
   b. Score: Each safe exercise gets a composite score based on:
      - Primary condition relevance × weight (doubled for primary)
      - Secondary condition relevance (averaged)
      - Mode adjustments:
        * Flare: +2 for breath/release, -2 for stability
        * Easier: +1 for breath, +0.5 for release
      - Stress/fibro/sleep bonus: +1 for breath/release
      - Anti-repetition penalty: -2 for recently used exercises
      - No-equipment bonus: +0.3
   c. Bucket: Exercises sorted into category buckets (breath, mobility, stability, release)
   d. Shuffle: Top half of each bucket is deterministically shuffled (seeded by week + conditions)
   e. Balance: Category targets enforced based on mode and exercise count

2. CATEGORY BALANCE TARGETS:
   Normal mode:
     breath=1, stability≥2 (25%), release≥1 (20%), mobility=remainder(≥2)
     (sleep_issues: release boosted to ≥2, 30%)

   Easier mode:
     breath=1, stability≥1 (15%), release≥1 (35%), mobility=remainder(≥1)

   Flare mode:
     breath=1, stability≥0 (10%), release≥1 (45%), mobility=remainder(≥0)

3. EXERCISE COUNT BY DURATION:
   ≤10 min → 3 exercises
   ≤15 min → 5
   ≤20 min → 6
   ≤30 min → 8
   ≤45 min → 10
   >45 min → 12

4. MODE DETERMINATION:
   Input: profile.flareToday, energyLevel, pain, fatigue, sleep, flareNow
   Rules:
   - flareToday=true OR flareNow="yes" → FLARE
   - pain≥7 OR fatigue≥7 OR sleep≤3 → FLARE
   - pain≥5 OR fatigue≥5 → EASIER
   - energyLevel="low" → EASIER
   - Otherwise → NORMAL

5. SESSION GENERATION:
   - Day distribution per sessions/week: {2:[2,5], 3:[2,4,6], 4:[1,3,5,7], 5:[1,2,4,6,7]}
   - All sessions in a plan share the same exercise selection
   - Session IDs: session_{timestamp}_{index}
   - Plan ID: plan_{timestamp}

6. ADAPTIVE CHECK-IN:
   After each session, if "too much" is true OR pain delta ≥ 2:
   - Next planned session switches to flare (if flareToday) or easier mode
   - New exercise selection generated for that session

7. SLEEP-SPECIFIC ORDERING:
   If user has sleep_issues:
   - Release exercises are placed at the END of the session order
   - At least 2 release exercises pushed to final positions

8. ANTI-REPETITION:
   - Last 20 used exercise IDs stored in localStorage (key: yogacareRecentExercises)
   - Recently used exercises get -2 score penalty
   - Deterministic seeded shuffle provides weekly variety

9. FIBROMYALGIA LEGACY POOLS:
   For fibro-only users, hardcoded exercise ID pools exist as fallback:
   - Flare: fib_breath_01, fib_breath_02, fib_mob_01, fib_rel_01, fib_rel_02, fib_rel_04, fib_mob_03
   - Easier: above + fib_mob_02, fib_stab_03
   - Normal: above + fib_stab_01, fib_stab_02

QUESTIONNAIRE BRANCHING

Entry: User selects conditions on /conditions page

Branch 1: Fibromyalgia selected
  → 3-step fibro-specific questionnaire
  → Step 1: pain, fatigue, sleep, flare, pain areas
  → Step 2: triggers, gentle movement effect, touch sensitivity, day type, activities
  → Step 3: equipment, restrictions, red flags

Branch 2: Any specific condition selected (not fibro-only)
  → 3-step condition-aware questionnaire
  → Step 1: pain, fatigue, sleep, flare
  → Step 2: Condition-specific panels (back, neck, shoulder, knee, hip, disc, sciatica, OA, pregnancy, postpartum, menopause, cross-training, stress, sleep) + daily routine
  → Step 3: equipment, restrictions, red flags

Branch 3: No recognized conditions
  → 3-step generic questionnaire
  → Step 1: main concern (text), pain slider
  → Step 2: limitations (text)
  → Step 3: equipment, red flags

RED FLAG ROUTING:
  If any red flag is checked → redirect to /stop page
  Red flags: "New sharp pain", "Increasing numbness or weakness",
  "Unusual dizziness or fainting", "Shortness of breath", "Fever or acute illness"
